What is this

Bitcoin Core fills blocks by picking transactions one at a time, highest fee-rate first. It misses a common pattern: a cheap transaction paired with an expensive one — where taking both together earns more than skipping them both.

This API sees those pairs. It packs your block with more of the right transactions, earning you more sats per block — automatically, with no changes to your node or hardware.

Median result on live mainnet: +578 bps more fees than Bitcoin Core's default. On a block with 0.3 BTC in fees, that's an extra ~$1,700. Per block. Every result is cryptographically signed so you can verify it yourself.

How to use it

Make one POST request

Replace your getblocktemplate call with this. Works on Linux, Mac, or Windows.

# Linux / Mac
curl -X POST https://nativebtc.org/v1/template/optimize \
  -H "Content-Type: application/json" \
  -d '{"use_optimizer": true}'
# Windows PowerShell
Invoke-RestMethod -Uri "https://nativebtc.org/v1/template/optimize" `
  -Method POST -ContentType "application/json" `
  -Body '{"use_optimizer": true}'
Get back a complete optimized template

The response includes your optimized block, a comparison against the Core baseline, your uplift in basis points, and a signed receipt.

Use the template in your Stratum server

The optimized transaction list is in the same format as Bitcoin Core's GBT. Feed it directly into your block builder. No format conversion needed.

Verify the signed receipt (optional but recommended)

Every response is signed with Ed25519. You can verify the signature independently — no need to trust our numbers.

Request options

Field Default What it does
use_optimizer true Set to false to get the Core baseline only (for comparison)
min_fee_rate_sat_vb 1.0 Ignore transactions below this fee rate
weight_limit 3,996,000 Target block weight (leaves room for coinbase)
max_tx_count 4,000 Maximum transactions in the template
